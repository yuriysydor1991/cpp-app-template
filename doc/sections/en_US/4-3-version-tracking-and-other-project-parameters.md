## Version tracking and other project parameters

The project's `meson.build` files are tracking the current project git repository commit, project name, configure date and project version that are forwarded into the project's configure header file located at `src/app/project-global-decls.h.in`. The `project-global-decls.h.in` will be compiled into the build directory as `project-global-decls.h` by the `configure_file()` call in `meson/template-project-configure-file/meson.build` during the project configure stage. The `ApplicationVersionPrinter` class includes this project's configure file and uses the information provided to print appropriate version info. Usually, this code is executed by calling the generated executable with a `-v` or `--version` command line flag. After the version is printed, the default implementation causes the application's `ApplicationVersionPrinter` instance to return zero value and causes the executable to stop it's execution.
