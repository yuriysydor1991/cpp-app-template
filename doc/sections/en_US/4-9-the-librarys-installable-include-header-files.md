## The library's installable public include header files

The [src/lib/facade/public](/src/lib/facade/public) directory contains installable public include header files. Public header files define the implemented library's main facade interface for the underlying functionality. Introduce new classes and/or methods there to share more functionality with any project of interest. The contents of [src/lib/facade/public](/src/lib/facade/public) will be installed for example under the default `include/CppAppTemplate-0` sub-directory (the name follows the project name and its major version) with respect to the configured Meson `--prefix`.

Alongside the successful installation of the library binary and header files, Meson will install:

- a **pkg-config file** at `<prefix>/<libdir>/pkgconfig/CppAppTemplate-0.pc`, so downstream Meson projects can probe the library with `dependency('CppAppTemplate-0')` (which uses pkg-config under the hood) and downstream Autotools or generic build systems can use it directly via `pkg-config --cflags --libs CppAppTemplate-0`.
- a set of **CMake package-config files** at `<prefix>/<libdir>/cmake/CppAppTemplate-0/`:
    - `CppAppTemplate-0Config.cmake`
    - `CppAppTemplate-0ConfigVersion.cmake` (SameMajorVersion policy)
    - `CppAppTemplate-0Targets.cmake` (exporting the imported target `CppAppTemplate-0::CppAppTemplate-0`)

Downstream CMake projects can then use the library with:

```
find_package(CppAppTemplate-0 CONFIG REQUIRED)
target_link_libraries(my_app PRIVATE CppAppTemplate-0::CppAppTemplate-0)
```

Both artefacts are generated by Meson `configure_file` calls in [src/lib/meson.build](/src/lib/meson.build) from templates that live in [src/lib/cmake/](/src/lib/cmake/). Adjust the templates and the `pkgconfig.generate()` invocation if the library needs additional metadata (extra `Requires:` for pkg-config, extra `find_dependency()` for CMake, etc.).
