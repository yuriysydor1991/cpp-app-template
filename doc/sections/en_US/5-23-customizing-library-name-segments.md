## Customizing the installable library name segments

The `applib` branch ships a main application binary **and** a separate shared
library that downstream consumers can link against. The library name they see
(the library binary, the `include/<name>/` subdir for public headers, the CMake
package directory and the imported-target namespace exported through
`install(EXPORT ...)`) is derived from three optional CMake options. They let
parallel installs of distinct versions of the library coexist on the same host -
e.g. `include/CppAppTemplate-0.11.0-dev/` next to `include/CppAppTemplate-0/`.
The main application executable name is unaffected and keeps using
`PROJECT_BINARY_NAME`.

| Option | Default | Effect |
|---|---|---|
| `-DLIB_INCLUDE_MINOR_IN_NAME=ON` | `OFF` | Appends `.<minor>` to the library name |
| `-DLIB_INCLUDE_MICRO_IN_NAME=ON` | `OFF` | Appends `.<micro>` (implies the minor flag) |
| `-DLIB_NAME_SUFFIX=-dev` | `""` | Appends an arbitrary trailing tag |

Resulting library name examples for a `0.11.0` project:

| Configure flags | Library name |
|---|---|
| (none) | `CppAppTemplate-0` |
| `-DLIB_INCLUDE_MINOR_IN_NAME=ON` | `CppAppTemplate-0.11` |
| `-DLIB_INCLUDE_MINOR_IN_NAME=ON -DLIB_INCLUDE_MICRO_IN_NAME=ON` | `CppAppTemplate-0.11.0` |
| `-DLIB_NAME_SUFFIX=-dev` | `CppAppTemplate-0-dev` |
| `-DLIB_INCLUDE_MINOR_IN_NAME=ON -DLIB_INCLUDE_MICRO_IN_NAME=ON -DLIB_NAME_SUFFIX=-dev` | `CppAppTemplate-0.11.0-dev` |

Combined configure example:

```
# inside the project root directory

cmake -S . -B build \
  -DLIB_INCLUDE_MINOR_IN_NAME=ON \
  -DLIB_INCLUDE_MICRO_IN_NAME=ON \
  -DLIB_NAME_SUFFIX=-dev
cmake --build build
cmake --install build --prefix /usr/local
```

The same name is used consistently for every installed library artefact:

- `<libdir>/lib<NAME>.so.<version>` (the shared library binary),
- `<prefix>/include/<NAME>/*.h` (the public headers install subdir, declared via the `INSTALL_INTERFACE` of the library target),
- `<libdir>/cmake/<NAME>/` + the `<NAME>::<NAME>` imported target generated by `install(EXPORT ...)` - downstream CMake projects use `find_package(<NAME> CONFIG)`.

The main application executable (`PROJECT_BINARY_NAME`, default `CppAppTemplate`) is unaffected by these options and continues to install under `<prefix>/<PROJECT_BINARY_INSTALLATION_DIR>/`.

The derivation lives in [cmake/template-project-misc-variables-declare.cmake](/cmake/template-project-misc-variables-declare.cmake) and is exposed as the `PROJECT_LIBRARY_NAME` CMake cache entry that the rest of the build system consumes. Enabling `LIB_INCLUDE_MICRO_IN_NAME` without `LIB_INCLUDE_MINOR_IN_NAME` triggers a configure-time warning and the minor flag is enabled implicitly to keep the produced name syntactically valid.
