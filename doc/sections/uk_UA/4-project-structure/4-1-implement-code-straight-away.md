## Реалізуй код одразу!

Щоб одразу приступити до реалізації конкретної програми необхідно перейти до файлу `LibMain.cpp` відповідного класу, який призначений для утримання початкового коду реалізації програми. Більш точно новий код повинен бути розміщений у методі `bool LibMain::libcall(std::shared_ptr<LibraryContext> ctx)`.

Хоча примірник класу `Application` що викликає метод `libcall` з примірника класу `LibMain` являється повністю реалізовани і не вимагає змін, його все-одно можна доповнити кодом для кращої відповідною роботи за необхідності. Більш детально пожна перегляну і відповідно доповнити у реалізації методу `int Application::run(std::shared_ptr<ApplicationContext> ctx)` класу `Application`.

Код класу `LibMain` буде скомпільовано у цьову бібліотеку для того щоб зробити його реалізацію функцій доступними для інших зовнішних програм.

**Але пам'ятаємо про принципи хорошого коду SOLID і його декомпозицію!**

Рекомендується створити нові окремі директорії у проекті котрі будуть розміщувати додаткові реалізовані компоненти програми і включити їх до реалізації класу `LibMain`. На противагу розміщення усього коду тільки всередині класу `LibMain` (для тривіального проекту це може бути оправданим).
