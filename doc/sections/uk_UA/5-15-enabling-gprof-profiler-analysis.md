## Вмикання підтримки профілювання з gprof

Для того щоб увімкнути підтримку профілювання за допомогою `gprof` необхідно встановити значення `true` для Meson-опції `ENABLE_GPROF` у Debug-побудові:

```
# всередині кореневої директорії проекту

meson setup build --buildtype=debug -DENABLE_GPROF=true
```

Після конфігурації проекту з увімкненою підтримкою `gprof` необхідно побудувати ціль `gprof-analyze`:

```
# всередині кореневої директорії проекту
# може не спрацювати !!!

meson compile -C build gprof-analyze
```

Дана команда у свою чергу побудує і запустить цільовий бінарний виконуваний файл проекту для генерації файлу `gmon.out` у директорії побудови (наприклад, `build/src/gmon.out`). Після чого Meson-скрипт запустить програму `gprof` для генерації цільового файлу аналізу `gprof-analyze.txt` який буде розміщений у директорії побудови проекту.

Профілювання за допомогою програми gprof може бути здійснене тільки у режимі побудови `--buildtype=debug`. Meson-підмодуль `meson/template-project-profiler-gprof/meson.build` повертає явну помилку якщо проект сконфігуровано з іншим типом побудови.
