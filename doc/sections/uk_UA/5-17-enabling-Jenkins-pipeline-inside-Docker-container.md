## Вмикання конвеєра Jenkins всередині Docker контейнера

**На даний момент ціль Jenkins-конвеєра у Docker контейнері не інтегровано у гілку `appMeson`.** CMake-ціль `jenkins-pipeline-docker-run` доступна лише у гілці `app`. Нотатки нижче описують очікуваний робочий процес після появи відповідної Meson-цілі; наразі скрипт конвеєра [misc/Jenkinsfile](/misc/Jenkinsfile) можна підключати у Jenkins, який запущено самостійно поза цим проектом.

Для того щоб увімкнути підтримку конвеєра [Jenkins](https://www.jenkins.io/) всередині спеціально виділеного Docker контейнера необхідно побудувати ціль `jenkins-pipeline-docker-run` за допомогою команд:

```
# всередині кореневої директорії проекту

# Конфігурування проекту для підтримки цілей Docker разом з Jenkins
meson setup build --buildtype=debug -DENABLE_DOCKER=true -DENABLE_JENKINS_DOCKER_PIPELINE=true

# Побудова і запуск контейнера для конвеєрів Jenkins
# або його перезапуск
meson compile -C build jenkins-pipeline-docker-run
```

Після чого необхідно знайти і запамʼятати пароль до [Jenkins](https://www.jenkins.io/) Web-інтерфейсу у логах першого запуску Jenkins системи, який може виглядати наступним чином:

```
# ...

*************************************************************
*************************************************************
*************************************************************

Jenkins initial setup is required. An admin user has been created and a password generated.
Please use the following password to proceed to installation:

aefdc6a4a2214c298c821236c739165c

This may also be found at: /var/jenkins_home/secrets/initialAdminPassword

*************************************************************
*************************************************************
*************************************************************

# ...
```

Необхідно відвідати локальну адресу `http://127.0.0.1:8080` (порт може змінюватись) за допомогою використовуваного браузера щоб завершити початкову інсталяцію і конфігурацію [Jenkins](https://www.jenkins.io/) системи на Web-сторінках його GUI.

Після успішної інсталяції і конфігурації [Jenkins](https://www.jenkins.io/) необхідно створити відповідний конвеєр перевірки проекту всередині Web-панелі [Jenkins](https://www.jenkins.io/).

Після інтеграції Meson-скрипти побудови контейнера для конвеєрів [Jenkins](https://www.jenkins.io/) очікувано будуть розташовані поруч з іншими Docker-цілями у директорії `meson/template-project-dockerers/`. Відповідний базовий примірник скрипта конвеєра перевірки шаблонного проекту розміщений за відносним шляхом [misc/Jenkinsfile](/misc/Jenkinsfile) котрий повинен бути вказаним для конфігурації конвеєра побудови у Web-панелі [Jenkins](https://www.jenkins.io/). Порт для Web-доступу контейнера Docker до конфігурації конвеєрів перевірки [Jenkins](https://www.jenkins.io/) може бути змінено за допомогою встановлення нового значення для Meson-опції `JENKINS_PIPELINES_PANEL_HTTP_PORT`.

Очікувано Meson-скрипти побудови Docker контейнера для конвеєрів перевірки [Jenkins](https://www.jenkins.io/) за замовчуванням перезапускатимуть попередньо побудований контейнер якщо ціль було запущено повторно - повторюючи поведінку CMake-реалізації з гілки `app`.
