## Вмикання підтримки генерування WIX MSI-пакетів з cpack

Для того щоб мати змогу згенерувати інсталяційний файл `.msi` для Microsoft Windows необхідно сконфігурувати проект для підтримки команди `cpack` за допомогою змінної `ENABLE_WIX`:

```
# всередині кореневої директорії проекту

cmake -B build -S . -DCMAKE_BUILD_TYPE=Release -DENABLE_WIX=ON
```

Наступним кроком буде збирання або компіляція усіх необхідних файлів які повинні входити у `.msi`-пакет:

```
# всередині кореневої директорії проекту

cmake --build build
```

На кінець, необхідно виконати команду `cpack` всередині директорії побудови проекту-шаблону:

```
# всередині директорії побудови проекту-шаблону

cpack -C Release
```

Згенерований пакет повинен з'явитись всередині кореня директорії побудови проекту. Наприклад, якщо ім'я проекту не змінювалось і його версія `0.12.0`, тоді назва пакету може виглядати наступним чином: `CppAppTemplate-0.12.0-win64.msi` (точний суфікс архітектури залежить від хоста, на якому виконується збирання).

Для того щоб переглянути або змінити параметри створення WIX MSI-пакетів необхідно відвідати файл [cmake/enablers/packagers/template-project-wix-enabler.cmake](/cmake/enablers/packagers/template-project-wix-enabler.cmake).

Інтеграція використовує виключно `cmake` і `cpack` - жодних сторонніх програм CMake-модуль проекту не залучає. Вбудований у CPack генератор `WIX` сам збирає `.msi`-пакет усередині, так само як генератор DEB збирає `.deb`-файли.

Опціонально можна перевизначити наступні кеш-змінні для налаштування згенерованого інсталятора:

- `TEMPLATE_PROJECT_WIX_UPGRADE_GUID` - стабільний GUID, що однозначно ідентифікує продукт між версіями (за замовчуванням: тимчасовий `12345678-1234-1234-1234-123456789012`). **Для реального випуску цей GUID необхідно замінити на унікальне значення проекту**, інакше виявлення оновлень буде колізіонувати з іншими проектами, які досі використовують це тимчасове значення. Згенерувати новий GUID можна командою PowerShell `[guid]::NewGuid()` або будь-яким генератором GUID.
- `TEMPLATE_PROJECT_WIX_PROGRAM_MENU_FOLDER` - назва теки у меню Пуск Windows, у якій інсталятор створює ярлики (за замовчуванням: назва проекту).
- `TEMPLATE_PROJECT_WIX_CULTURES` - перелік UI-локалей інсталятора, розділених крапкою з комою (за замовчуванням: `en-US`, наприклад `en-US;uk-UA`).
- `TEMPLATE_PROJECT_WIX_LICENSE_RTF` - опціональний абсолютний шлях до файлу RTF або TXT з ліцензією, який буде показано інсталятором.
- `TEMPLATE_PROJECT_WIX_PRODUCT_ICON` - опціональний абсолютний шлях до файлу `.ico`, який використовується як піктограма продукту у MSI.

Окрім цього, видавця пакету (`vendor`) береться з `PROJECT_MAINTAINER`, контактну адресу - з `PROJECT_MAINTAINER_EMAIL`, опис - з `DESCRIPTION` проекту. Корінь встановлення під `Program Files` - це `CMAKE_PROJECT_NAME`. Жодних додаткових оголошень у CMake-файлах проекту вносити не потрібно - достатньо просто увімкнути опцію.

> **Зауваження щодо запуску.** Подібно до генератора `DEB`, який делегує роботу `dpkg-deb`, генератор `WIX` у CPack делегує роботу [WiX Toolset](https://wixtoolset.org/) (інструменти `candle.exe` / `light.exe`). Тому пакування очікувано виконується на Microsoft Windows зі встановленим WiX Toolset, доступним з `PATH`. Сам CMake-модуль є платформо-незалежним - він лише налаштовує CPack і не викликає жодних сторонніх програм безпосередньо.
