## Побудова документації

На даний момент доступна побудова документації за допомогою програми Doxygen з наявних коментарів вихідного джерельного коду проекту.

Для того щоб зробити доступною ціль для побудови документації за допомогою Doxygen під час його конфігурації необхідно встановити Meson-опцію `ENABLE_DOC_DOXYGEN` у значення `true` (для ОС на базі GNU/Linux):

```
# з кореневої директорії проекту-шаблону

meson setup build -DENABLE_DOC_DOXYGEN=true
```

Яка в свою чергу створить директорію з назвою `build` всередині кореневої директорію проекту-шаблону і сконфігурує проект з увімкненою ціллю для побудови Doxygen документації.

На кінець можна побудувати документацію виконуючи команду:

```
# всередині кореневої директорії проекту

meson compile -C build Doxygen-doc
```

Яка у свою чергу створює директорію `doc/CppAppTemplate-html` (вже додана до файлу `.gitignore`) що буде містити зібрану документацію у вигляді HTML-сторінок і ресурсів до них. Для того щоб відкрити і оглянути згенеровану документацію необхідно відкрити файл який лежить за шляхом `doc/CppAppTemplate-html/index.html` від кореня проекту. Директорія `CppAppTemplate-html` змінить назву у відповідності до зміни назви проекту шаблону (перший позиційний аргумент `project(...)` у кореневому файлі `meson.build`) або змінної `DOXYGEN_OUT_HTML_NAME` яка визначає ім'я для цілої цільової директорії для згенерованої HTML-документації.

Файл `doc/Doxyfile.meson.in` від кореня проекту містить усі параметри конфігурації Doxygen, які можуть бути зміненими для зміни виводу документації.
