## Вмикання підтримки статичного аналізатора коду clang-tidy

Для того щоб увімкнути статичну перевірку коду кожної одиниці транслювання або ж кожного файлу `*.cpp` у проекті за допомогою програми `clang-tidy` необхідно увімкнути її під час конфігурування системи побудови проекту, встановлюючи опцію `ENABLE_CLANG_TIDY` у значення `true`:

```
# всередині кореневої директорії проекту-шаблону

meson setup build -DENABLE_CLANG_TIDY=true
```

Щоб виконати статичний аналіз коду необхідно запустити побудову Meson-цілі `clang-tidy` - команда `clang-tidy` автоматично перевірить кожен джерельний файл:

```
# всередині кореневої директорії проекту

meson compile -C build clang-tidy
```

Щоб переглянути докладну інформацію по статичному аналізатору `clang-tidy` необхідно переглянути файли `meson/template-project-clang-tidy-target/meson.build`, `misc/.clang-tidy` і документацію до відповідної використовуваної версії `clang-tidy`.

**Попередження** Виконання статичного аналізу коду за допомогою `clang-tidy` може значно збільшити час побудови цілей.

Деталі встановлення `clang-tidy`: [Необов'язкові пакети для статичного аналізатора коду clang-tidy](/doc/sections/uk_UA/3-7-optional-for-the-code-analyzer-with-clang-tidy.md).
