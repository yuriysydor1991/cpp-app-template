## Вмикання цілі динамічної перевірки пам'яті Valgrind

Для того щоб увімкнути для побудови ціль `valgrind` необхідно сконфігурувати проект з увімкненою опцією `ENABLE_VALGRIND`:

```
# всередині кореневої директорії проекту

meson setup build -DENABLE_VALGRIND=true
```

Для виконання динамічної перевірки пам'яті у виконуваного файлу проекту необхідно виконати наступну команду:

```
# всередині кореневої директорії проекту

meson compile -C build valgrind
```

Деталі цілі `valgrind` можуть бути переглянутими і/або зміненими у файлі `src/meson.build` (гілка `ENABLE_VALGRIND`). Ціль запускає `valgrind --leak-check=full --error-exitcode=1 --show-leak-kinds=all` на головному виконуваному файлі проекту.

**Попередження! Ціль запустить на виконання головний виконуваний файл проекту.** Динамічна перевірка пам'яті виконуваного файлу проекту-шаблону вимагає запуск програми і повний прохід виконання. Необхідно забезпечити кінечність виконання програми щоб запобігти невизначено довгий час перевірки.

Деталі встановлення `valgrind`: [Необов'язкові пакети для перевірки використання пам'яті за допомогою Valgrind](/doc/sections/uk_UA/3-8-optional-for-the-memory-checkwith-Valgrind.md).
