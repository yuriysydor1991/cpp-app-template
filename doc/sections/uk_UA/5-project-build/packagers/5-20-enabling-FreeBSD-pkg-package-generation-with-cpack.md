## Вмикання підтримки генерування пакунків FreeBSD pkg з cpack

Для того щоб мати змогу згенерувати бінарний пакунок FreeBSD `.pkg` необхідно сконфігурувати проект для підтримки команди `cpack` за допомогою змінної `ENABLE_FREEBSD_PKG`:

```
# всередині кореневої директорії проекту 

mkdir -vp build && cd build && cmake ../ -DENABLE_FREEBSD_PKG=ON
```

Наступним кроком буде збирання або компіляція усіх необхідних файлів які повинні входити у FreeBSD `pkg`-пакунок:

```
# всередині директорії побудови проекту-шаблону

cmake --build . --target all
```

На кінець, необхідно виконати команду `cpack` всередині директорії побудови проекту-шаблону:

```
# всередині директорії побудови проекту-шаблону

cpack
```

Згенерований пакунок повинен з'явитись всередині кореня директорії побудови проекту. Наприклад, якщо ім'я проекту не змінювалось і його версія `0.12.0`, тоді назва пакунку може виглядати наступним чином: `CppAppTemplate-0.12.0-amd64.pkg` (точний суфікс архітектури залежить від хоста, на якому виконується збирання).

Для того щоб переглянути або змінити параметри створення FreeBSD `pkg`-пакунків необхідно відвідати файл [cmake/enablers/packagers/template-project-freebsd-pkg-enabler.cmake](/cmake/enablers/packagers/template-project-freebsd-pkg-enabler.cmake).

Інтеграція використовує виключно `cmake` і `cpack` - жодних сторонніх програм CMake-модуль проекту не залучає. Вбудований у CPack генератор `FREEBSD` сам збирає бінарний пакунок усередині, так само як генератор DEB збирає `.deb`-файли.

Опціонально можна перевизначити наступні кеш-змінні для налаштування метаданих згенерованого пакунку:

- `TEMPLATE_PROJECT_FREEBSD_PKG_LICENSE` - SPDX-подібний тег ліцензії (за замовчуванням: `BSD2CLAUSE`). Поширені значення: `BSD2CLAUSE`, `BSD3CLAUSE`, `GPLv3`, `MIT`, `APACHE20`.
- `TEMPLATE_PROJECT_FREEBSD_PKG_ORIGIN` - походження порту FreeBSD у формі `<категорія>/<назва>` (за замовчуванням: `misc/<імʼя-бінарника-в-нижньому-регістрі>`).
- `TEMPLATE_PROJECT_FREEBSD_PKG_CATEGORIES` - перелік категорій `pkg`, розділених крапкою з комою (за замовчуванням: `misc`).

Окрім цього, поле `WWW` пакунку береться з `HOMEPAGE_URL`, оголошеного у верхньорівневому `CMakeLists.txt`, електронна адреса супровідника - з `PROJECT_MAINTAINER_EMAIL`, а коментар та опис - з `DESCRIPTION` проекту. Жодних додаткових оголошень у CMake-файлах проекту вносити не потрібно - достатньо просто увімкнути опцію.

> **Зауваження щодо запуску.** Подібно до генератора `DEB`, який делегує роботу `dpkg-deb`, генератор `FREEBSD` у CPack покладається на `libpkg` з системи `pkg` FreeBSD. Тому пакування очікувано виконується на FreeBSD (або на Linux-хості, де встановлена збірка CMake з підтримкою `libpkg`). Сам CMake-модуль є платформо-незалежним - він лише налаштовує CPack і не викликає жодних сторонніх програм безпосередньо.
