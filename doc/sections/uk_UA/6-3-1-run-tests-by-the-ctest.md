### Запуск тестів через meson test

Якщо розробник увімкнув цілі побудови тестів за допомогою опції `ENABLE_UNIT_TESTS` чи `ENABLE_COMPONENT_TESTS` і після успішної їх побудови, можна пустити на виконання усі наявні тести з допомогою команди `meson test` з директорії побудови проекту:

```
# з директорії побудови проекту

meson test
```

Як альтернатива, можна виконати команду `meson test` з будь-якого розміщення у файловій системі за допомогою прапорця `-C` із зазначеним шляхом до директорії побудови (для GNU/Linux ОС і подібних):

```
meson test -C build
```

Для того щоб виконати окремий конкретний тест необхідно виконати команду `meson test` з назвою тесту вказаного після прапорця `--suite`. Наприклад, для юніт тесту `UTEST_ApplicationFactory` команда буде виглядати на щось подібне:

```
# всередині кореневої директорії проекту-шаблону

meson test -C build --suite UTEST_ApplicationFactory
```
