### Запуск тестів за допомогою ctest

Якщо розробник увімкнув цілі побудови тестів за допомогою змінної `ENABLE_UNIT_TESTS` і після успішної їх побудови, можна пустити на виконання усі наявні тести з допомогою команди `meson test` з директорії побудови проекту:

```
# з директорії побудови проекту

meson test
```

Як альтернатива, можна виконати команду `meson test` з будь-якого розміщення у файловій системі за допомогою абсолютного шляху до директорії побудови проекту-шаблону (для GNU/Linux ОС і подібних):

```
meson test -C /шлях/до/директорії/побудови/проекту
```

Для того щоб виконати окремий конкретний текст необхідно виконати команду `meson test` з назвою теста вказаного після прапорця `--suite`. Наприклад, для юніт тесту `UTEST_ApplicationFactory` команда буде виглядати на щось подібне:

```
# всередині директорії побудови проекту-шаблону

meson test -C build --suite UTEST_ApplicationFactory
```
