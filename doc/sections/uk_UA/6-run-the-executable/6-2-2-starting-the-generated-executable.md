### Запуск згенерованого виконуваного файлу

Якщо виконуваний бінарний файл скомпільований і наявний у директорії побудови проекту його можна запустити з терміналу використовуючи шлях знайдений з попередньої секції командою:

```
# наприклад, з директорії побудови

./src/CppAppTemplate --docroot ".;/resources" --http-listen 127.0.0.1:8000 --accesslog CppAppTemplate-http-access.log
```
 
Команда запустить Web-сервер побудованої [Wt](https://www.webtoolkit.eu/wt)-програми з реалізованою логікою Web-сайту котрий буде "слухати" запити на порт `8000` loopback мережевого інтерфейсу.

У разі необхідності очікування запитів з загального мережевого інтервейсу, себто ззовні локальної машини, необхідно замінити адресу `127.0.0.1` на `0.0.0.0`.

Вказана у команді запуску директорія `resources` повинна бути скопійована з використаного вихідного коду проекту [Wt](https://www.webtoolkit.eu/wt) у директорію розміщення і запуску виконуваного файлу веб-сайту для коректного відображення і функціонування сайту. Дана директорія ресурсів не включається в поточний шаблонний проект щоб зменшити розмір git репозиторію. Детальніше про те як отримати директорію `resources` у документації до [Wt C++ фреймворку](https://www.webtoolkit.eu/wt).

Щоб отримати детальний список параметрів командного рядка для виконуваного файлу побудованого разом з [Wt C++ фреймворком](https://www.webtoolkit.eu/wt) необхідно звернутись до його документації.
